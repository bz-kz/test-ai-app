# スキル、MCP、プラグイン

## エージェントmdの書き方
---
name: planner # カスタムエージェントの名前
description: This custom agent researches and plans new features for VS Code extensions. # エージェントの説明
model: opus # Claudeのモデル
tools: [execute, read, edit, search, web, agent, todo] # execute=コマンド実行,read=ファイル読む,edit=ファイル変数,search=PJ内のファイル検索(grepとかreadで全てのファイルを読むとトークン消費が激しいがsearchでファイルと記載場所を特定してからアクセスするとトークン節約できる。どちらを使ったほうが効率的かはAIが判断する),web=web検索,agent=agent呼び出し,todo=todoリストの作成、更新、完了状態を記録管理。plannerではtools: executeは不要。
handoffs:
  - label: Start Implementation # UI上のボタン
    agent: generator # どのエージェントに渡すか
    prompt: Implement the plan # どのような指示を渡すか
    send: true # 自動で渡すか、ユーザーの確認を待つか
    model: sonnet # どのモデルを使うか
    when: when the plan is complete and ready for implementation # どのタイミングで渡すか
---


## とりあえず入れておきたいプラグイン
1. Playwright => ブラウザ自動操作
2. GitHub => GitHub操作の自動化
3. 言語別LSP => リアルタイムコード解析
4. Superpowers => 能力を大幅に強化するスキル群(ベストプラティクスを意識した動作にしてくれる)
5. Context7 => 最新ドキュメント参照(学習データが古い問題を解決)
6. Code Review => コードレビューの自動化（レビュアーサブエージェントが動いてくれる）
7. Frontend Design => UIデザイン品質を底上げするスキル群(入れておくだけでいい。AIっぽいUIを避けれる。DESIGN.mdと併用が推奨)
8. Slack => Claudeへの指示

## rules の活用
- `.claude/rules/` にmdファイルを作ると、claudeが必要なときだけ読みにいくファイルとしてCLAUDE.mdを切り出せる。このファイルは毎回読まないので、コンテキストを節約できる

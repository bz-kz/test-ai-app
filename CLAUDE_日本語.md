# プロジェクト: AI 医療記録ジェネレーター (ローカル LLM)

## 1. システムアーキテクチャとハンドオフ
このプロジェクトは、`.claude/agents/` で定義された 3 エージェント ハーネスを使用します。
-**プランナー (Opus):**高レベルのデザインと範囲。 `SPEC.md`を更新します。
-**ジェネレーター (ソネット):**スプリントでの実装。 「TASKS.md」を更新します。
-**評価者 (Opus):**厳格な QA とレビュー。自己評価のバイアスを防ぎます。

## 2. コンテキストと技術スタック
-**モノレポ:**`/frontend` (Next.js 15 以降、TS、Tailwind)、`/backend` (Python 3.12 以降、FastAPI)、`/docs`。
-**データベース:**PostgreSQL。
-**言語:**ヒューマン チャット (JP)、ドキュメント (英語) (トークン節約)、コード コメント (JP)。

## 3. 基本原則
-**効率:**最初にコードを表示します。最小限の散文。 `/model opus` は計画/複雑なリファクタリングの場合にのみ使用してください。
-**信頼性:**厳密な型安全性 (「いかなる」もなし)。 「SPEC.md」に従ってドメイン駆動設計 (DDD) を使用します。
-**ハーネス フロー:**1. `SPEC.md` と `TASKS.md` を確認します。
  2. タスクを実行します。
  3. 「npm run build」または「pytest」を実行して確認します。
  4. 「TASKS.md」を「DONE」としてマークします。
## 4. コマンド
-**フロントエンド:**`cd フロントエンド && npm run dev` | 「npm 実行ビルド」
-**バックエンド:**`cd バックエンド && uvicorn main:app --reload` | 「pytest」
-**Git:**セッション開始時の「git log --oneline -n 5」は状態を同期します。

## 5. コーディング標準
### フロントエンド (Atomic Design + Onion)
-コンポーネント: `frontend/src/components/{原子|分子|生物}`。
-ロジック: API 呼び出しを `services/` または `hooks/` に分離します。
-定数: ハードコーディングする前に `frontend/src/lib/constants.ts` を確認してください。

### バックエンド (DDD + FastAPI)
-構造: `app/domain`、`app/usecases`、`app/インフラストラクチャ`、`app/interfaces`。
-検証: すべてのリクエスト/レスポンスボディに Pydantic モデルを使用します。

## 6. 完了の定義 (国防総省)
-[ ] コードは、定義された Onion/DDD アーキテクチャに従います。
-[ ] lint/type エラーはありません (`tsc` または `pyright` が渡されました)。
-[ ] 評価エージェントが `SPEC.md` に対してロジックを検証しました。
-[ ] `TASKS.md` が更新され、簡潔な git コミットが作成されます。

## 7. サブエージェントとスキル
-役割固有のタスクには `.claude/agents/*.md` を使用します。
-共有スキルは `.claude/skills/` にあります。
